HTML DOM and Browser Support

  • Shapes can be rendered with three different browser engines: Canvas, SVG and VML.
  • If no special rendering engine is needed, the drawing canvas itself searches for the best engine available, Canvas, SVG, or VML, in this order.
  • In seldom cases it might be useful to force a drawing canvas to use a specific rendering engine. Rendering engines can be forced within the construction of a drawing canvas object, e.g., by calling new xrx.drawing.Drawing(element1, xrx.engine.CANVAS), new xrx.drawing.Drawing(element2, xrx.engine.SVG) or new xrx.drawing.Drawing(element3, xrx.engine.VML).
  • It is recommended to wrap a constructor call with a xrx.canvas.isSupported(), xrx.svg.isSupported() or xrx.vml.isSupported() condition if one forces a special rendering engine.

The SVG rendering engine gives nice insight into the internal mode of operation of a drawing canvas.

<svg height="300" width="540" style="overflow: hidden;">
  <g transform="matrix(1,0,0,1,0,0)"> <!-- view-box -->
    <g> <!-- background layer -->
      <image></image>
    </g>
    <g> <!-- shape-layer -->
      <polygon points="10,10 50,15 51,100 178,120 3,190 " stroke="#1919A3" stroke-color="#1919A3" stroke-width="4"></polygon>
    </g>
    <g></g> <!-- shape-modify-layer -->
    <g></g>  <!-- shape-create-layer -->
  </g>
  <g></g>  <!-- tool-layer -->
  <polygon points="0,0 540,0 540,300 0,300 " fill="" fill-opacity="0" stroke="black" stroke-color="black" stroke-width="0"></polygon>
</svg>
  • The first child element of a drawing canvas is a graphic group <g> representing the view-box of a drawing canvas. The view-box group has a transformation matrix attached that is used to realize pan-zoom-rotate functionality.
  • Inside the view-box group there are four different layers: background-layer, shape-layer, shape-modify-layer, and shape-create-layer. holds all shapes that shall be rendered on the drawing canvas.
    • The background-layer is useful to realize annotation functionality with a drawing canvas. Annotation tools aim to draw shapes not just on a blank paper but in front of an image, PDF document or a video, for example. Currently, only images are supported for the background-layer.

This mode of operation is the same for all rendering engines, besides that VML rendering uses different DOM elements and the Canvas rendering engine has only one DOM element <canvas> whilst all shapes are rendered on the pixel-level without any DOM representation.
